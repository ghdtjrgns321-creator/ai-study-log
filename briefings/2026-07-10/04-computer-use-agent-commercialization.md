# 오늘의 AI 개념: 컴퓨터 사용 에이전트(Computer Use Agent)의 상용화

> 작성일: 2026-07-10 · 분류: trend

## 한 줄 정의

컴퓨터 사용 에이전트란 API나 별도 연동 없이도 사람처럼 화면을 보고 마우스·키보드를 조작해 웹사이트나 데스크톱 프로그램을 직접 다루는 AI 에이전트를 말한다.

## 쉬운 설명

기존 업무 자동화는 시스템이 제공하는 API나 정해진 연동 방식이 있어야 동작했다. API가 없는 프로그램이나 레거시 시스템은 자동화 대상에서 아예 제외되는 경우가 많았다.

컴퓨터 사용 에이전트는 이 전제를 뒤집는다. 화면을 스크린샷으로 찍어 무엇이 어디 있는지 시각적으로 파악한 다음, 사람처럼 클릭·입력·스크롤로 작업을 수행한다. 비유하면, 기존 자동화가 건물 뒷문에 미리 열쇠를 맞춰 둬야 들어갈 수 있는 방식이라면, 컴퓨터 사용 에이전트는 정문으로 걸어 들어가 사람처럼 문을 열고 버튼을 누르는 방식이다. 열쇠가 없는 곳도 갈 수 있지만, 그만큼 느리고 화면 변화에 취약하다.

브라우저 자동화 도구(Selenium, Playwright 등)와도 다르다. 기존 자동화 도구는 사람이 미리 정해 둔 스크립트를 그대로 재생하는 데 그치지만, 컴퓨터 사용 에이전트는 화면을 보고 그때그때 판단해 다음 행동을 스스로 정한다는 점에서 훨씬 유연하지만 예측 가능성은 떨어진다.

## 동작 원리

Claude Code의 컴퓨터 사용 기능을 예로 들면, 에이전트는 화면 조작이 필요한 작업을 만났을 때 가장 정밀한 도구부터 순서대로 시도한다. 해당 서비스의 MCP 서버가 있으면 그것을 쓰고, 셸 명령으로 되는 일이면 Bash를, 브라우저 작업이면 브라우저 연동 기능을 쓰고, 그 무엇도 안 되는 경우에만 화면을 직접 조작하는 컴퓨터 사용으로 넘어간다.

실행 과정에서는 화면 전체를 캡처해 모델이 읽기 쉬운 해상도로 축소한 뒤 전달하고, 모델은 이 이미지를 보고 클릭 좌표나 입력할 텍스트를 결정한다. 작업 도중에는 사용자가 승인한 앱만 조작할 수 있고, 셸·파일시스템급 권한을 가진 앱은 별도 경고와 함께 승인을 받도록 설계돼 있다.

## 구체 예시·사례

Claude Code CLI의 컴퓨터 사용 기능은 macOS에서 실행 중인 네이티브 앱을 빌드하고 실행한 뒤, 버튼을 하나씩 눌러가며 동작을 검증하고 오류 화면을 스크린샷으로 남기는 식으로 쓰인다. API가 없는 iOS 시뮬레이터나 디자인 툴처럼, 명령줄만으로는 다룰 수 없는 GUI 전용 도구를 다룰 때 특히 유용하다.

업계 전반에서는 웹 자동화 전용 오픈소스 프로젝트 브라우저 유즈(Browser Use)나, 브라우저 자체를 에이전트 실행 환경으로 재설계한 신생 서비스 Aside 같은 사례가 등장하며, 정해진 API 연동 없이도 웹 업무 전반을 대신 처리하는 방향으로 시장이 넓어지고 있다.

## 왜 지금 중요한가

Claude Code는 2026년 시점 공식 문서에서 macOS CLI에 컴퓨터 사용 기능을 정식으로 문서화해 뒀고, Pro·Max 플랜에서 연구 프리뷰로 제공한다고 명시한다. 셸 명령을 감싸는 샌드박싱과 달리 컴퓨터 사용은 실제 데스크톱에서 동작하므로 신뢰 경계가 다르다는 점도 함께 안내하고 있다.

업계 벤치마크 동향을 보면, 컴퓨터 조작 능력을 측정하는 OSWorld 벤치마크에서 Claude 계열 모델 점수가 출시 초기 10%대에서 2026년 상반기 기준 70%대 후반에서 80%대까지 상승했다는 보도가 다수 나오고 있다. 다만 매체마다 인용하는 정확한 수치와 모델 버전이 다소 엇갈려 보이므로, 특정 수치를 인용할 때는 출처와 시점을 함께 확인할 필요가 있다.

시장 조사 매체들은 2026년 상반기를 기점으로 거의 모든 주요 빅테크가 브라우저·컴퓨터 사용 에이전트를 소비자 제품이나 개발자 도구, 엔터프라이즈 API 형태로 내놓고 있다고 전하며, 실험 단계를 지나 상용 도입 단계로 접어들었다고 진단한다.

## 회계법인 AI 직무 연결 포인트

회계·감사 실무에는 API가 없는 레거시 회계 시스템이나 사내 전용 프로그램이 여전히 많다. 컴퓨터 사용 에이전트는 이런 시스템에서도 사람이 하듯 화면을 보고 데이터를 조회·입력할 수 있어, API 연동이 불가능했던 업무의 자동화 범위를 넓힐 잠재력이 있다.

다만 감사 증거 확보 관점에서는 신중해야 한다. 화면을 눈으로 보고 판단해 조작하는 방식은 클릭 하나의 오차가 잘못된 데이터 입력이나 승인으로 이어질 수 있어, 재무 데이터를 다루는 업무에 적용하려면 사람의 검토 단계와 로그 기록이 반드시 함께 있어야 한다. Claude Code 문서가 컴퓨터 사용을 가장 느리고 신뢰 경계가 다른 최후의 수단으로 규정하는 것도 같은 맥락이다.

면접에서는 "AI 에이전트가 회계 업무 자동화 범위를 어디까지 넓힐 수 있는가"라는 질문에, API 기반 자동화가 안 되는 영역까지 컴퓨터 사용 에이전트가 메울 수 있지만 그만큼 신뢰 경계와 검증 절차 설계가 함께 따라와야 한다는 균형 잡힌 답을 준비할 수 있다.

## 핵심 용어·논쟁

- 컴퓨터 사용(Computer Use) — AI가 스크린샷을 보고 마우스·키보드를 직접 조작해 화면 기반 작업을 수행하는 방식.
- OSWorld — 컴퓨터 사용 에이전트의 실제 운영체제 작업 수행 능력을 측정하는 대표적 벤치마크.
- 브라우저 에이전트(Browser Agent) — 웹 브라우저 환경에 특화된 컴퓨터 사용 에이전트.
- MCP(Model Context Protocol, 모델 컨텍스트 프로토콜) — AI가 외부 도구·데이터에 표준화된 방식으로 접근하게 하는 프로토콜.

논쟁거리는 두 가지로 보인다. 하나는 벤치마크 수치의 신뢰성으로, 매체마다 보고하는 OSWorld 점수 편차가 커서 실제 실무 신뢰도를 그대로 반영하는지 의문이 제기된다. 다른 하나는 안전성으로, 화면에 표시된 악성 콘텐츠가 에이전트를 속이는 프롬프트 인젝션 위험이 컴퓨터 사용 환경에서는 셸 명령보다 통제하기 어렵다는 지적이 나온다.

## 자료 깊이 읽기

### Let Claude use your computer from the CLI (Claude Code 공식 문서) — 영어, 문서, 중급

Claude Code CLI에 탑재된 컴퓨터 사용 기능의 공식 레퍼런스다. `/mcp`에서 `computer-use` 서버를 켜는 절차, macOS 손쉬운 사용·화면 기록 권한 승인 흐름, 세션당 한 번에 하나의 에이전트만 화면을 제어할 수 있다는 잠금 정책을 구체적으로 설명한다. 도구 우선순위(MCP → Bash → 브라우저 연동 → 컴퓨터 사용) 설계와, 터미널 창은 스크린샷에서 항상 제외해 에이전트가 자기 자신의 출력에 속지 않도록 한 안전장치가 인상적이다. 샌드박싱과 달리 컴퓨터 사용은 실제 데스크톱에서 동작해 신뢰 경계가 다르다는 점을 문서가 스스로 명시하고 있어, 도입 여부를 판단할 때 참고할 만하다.

### Computer Use and GUI Agents in 2026: State of the Art (Zylos Research) — 영어, 블로그, 중급

컴퓨터 사용·GUI 에이전트 분야의 2026년 현황을 정리한 리서치 글이다. 화면 이해 정확도가 2025년 말부터 크게 개선됐고 작업당 처리 비용이 낮아졌다는 흐름을 보고하지만, 이 글이 제시하는 구체적 OSWorld 점수는 다른 매체가 보고하는 Claude Opus 4.7·4.8 계열 수치와 차이가 있어 하나의 소스로만 판단하기는 어렵다. 그럼에도 컴퓨터 사용 에이전트가 연구 단계를 지나 비용 효율이 뒷받침되는 상용 단계로 넘어가고 있다는 방향성 자체는 다른 자료들과 일치한다.

## 자가 점검 질문

1. 컴퓨터 사용 에이전트가 기존 API 기반 자동화나 브라우저 스크립트 자동화와 근본적으로 다른 지점은 무엇인가.
2. 재무·회계 데이터를 다루는 업무에 컴퓨터 사용 에이전트를 적용한다면, 어떤 검증 절차를 반드시 함께 둬야 하는가.
3. OSWorld 같은 벤치마크 수치가 매체마다 엇갈릴 때, 실무 도입 판단에는 이를 어떻게 반영해야 하는가.
