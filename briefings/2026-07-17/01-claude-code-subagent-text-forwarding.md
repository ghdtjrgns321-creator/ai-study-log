# 오늘의 AI 개념: 서브에이전트 텍스트·사고 포워딩(Subagent Text Forwarding)

> 작성일: 2026-07-17 · 분류: claude-code

## 한 줄 정의

Claude Code를 자동화 스크립트로 실행할 때, 하위 작업을 맡은 서브에이전트가 속으로 무슨 생각을 했고 무슨 말을 했는지까지 로그로 뽑아볼 수 있게 하는 기능이다.

## 쉬운 설명

Claude Code는 복잡한 작업을 받으면 종종 하위 작업을 서브에이전트에게 위임한다. 이때 스크립트에서 `--output-format stream-json`으로 진행 상황을 실시간으로 받아보더라도, 지금까지는 서브에이전트가 어떤 도구를 호출했고 그 결과가 무엇인지(`tool_use`, `tool_result`)만 보였고, 서브에이전트가 그 판단에 이르기까지 무엇을 썼고 무슨 생각을 했는지(텍스트·thinking)는 보이지 않았다.

비유하면 팀장이 팀원에게 조사를 맡겼을 때, 팀원이 보낸 최종 보고서와 첨부 자료(도구 호출 결과)는 볼 수 있지만, 그 팀원이 조사 중 메모장에 적어둔 생각의 흐름은 볼 수 없던 상황과 같다. v2.1.211부터는 `--forward-subagent-text` 플래그나 `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 환경변수를 켜면 그 메모장까지 함께 넘겨받을 수 있다.

기존에도 메인 대화의 텍스트·thinking은 항상 스트림에 노출되어 있었다. 이번 변경의 핵심은 "메인 대화"와 "서브에이전트"에 적용되는 기본값이 달랐다는 점을 겨냥해, 서브에이전트에도 같은 수준의 가시성을 선택적으로 열어준 것이다.

## 동작 원리

stream-json 모드에서 서브에이전트가 보내는 모든 메시지에는 `parent_tool_use_id`라는 필드가 붙는다. 이 값이 해당 서브에이전트를 호출한 도구 호출의 ID와 같으면 그 메시지가 서브에이전트에서 나온 것이고, 메인 대화의 메시지는 이 필드가 `null`이다.

기본값에서는 이 서브에이전트발 메시지 중 `tool_use`·`tool_result` 블록만 스트림에 노출된다. `--forward-subagent-text`(또는 동일 기능의 환경변수)를 켜면 여기에 서브에이전트의 `text`·`thinking` 블록까지 추가로 노출되어, `parent_tool_use_id`로 묶어서 재구성하면 서브에이전트가 실제로 어떤 순서로 사고했는지 전체 트랜스크립트를 복원할 수 있다.

```bash
claude -p "코드베이스를 분석해 보고서를 작성해줘" \
  --output-format stream-json --verbose \
  --forward-subagent-text | \
  jq 'select(.parent_tool_use_id != null)'
```

이 명령은 메인 대화 메시지는 걸러내고, 특정 서브에이전트(`parent_tool_use_id`로 식별)가 만들어낸 텍스트·사고·도구 호출 이벤트만 추출한다.

## 비슷한 것과 비교

| 구분 | 기본값(플래그 미사용) | `--forward-subagent-text` 사용 |
|---|---|---|
| 서브에이전트 도구 호출(`tool_use`/`tool_result`) | 노출 | 노출 |
| 서브에이전트 텍스트·thinking | 숨김 | 노출 |
| 메인 대화 텍스트·thinking | 항상 노출 | 항상 노출 |
| 서브에이전트 전체 트랜스크립트 재구성 | 불가능 | `parent_tool_use_id`로 가능 |

선택 기준은 단순하다. 서브에이전트가 최종적으로 무엇을 했는지만 필요하면 기본값으로 충분하고, 서브에이전트가 왜 그렇게 판단했는지까지 디버깅·감사해야 한다면 플래그를 켠다. 다만 이 필드는 v2.1.211 이상에서만 지원된다.

## 왜 지금 중요한가

Claude Code 공식 changelog는 v2.1.211(2026-07-15 출시) 항목에 "`--forward-subagent-text` 플래그와 `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 환경변수를 추가해 stream-json 출력에 서브에이전트의 텍스트와 thinking을 포함하도록 했다"고 명시하고 있다.

공식 헤드리스 모드 문서 역시 이 필드를 상세히 설명하며, 이 기능이 필요해진 배경으로 "서브에이전트가 늘어나면서 최종 결과만으로는 중간 판단 과정을 재구성할 수 없다"는 관측성 공백을 든다. 멀티에이전트·다이나믹 워크플로우처럼 여러 서브에이전트가 동시에 작업을 나눠 맡는 구조가 늘어날수록, 각 서브에이전트의 사고 과정을 사후에 들여다볼 수 있는지가 자동화 파이프라인 신뢰성의 실질적 병목이 된다.

## 회계법인 AI 직무 연결 포인트

회계법인이 AI 에이전트를 감사·세무 업무에 투입할 때 가장 먼저 부딪히는 질문은 "이 결론에 왜 도달했는가"를 사후에 설명할 수 있느냐다. 서브에이전트가 여러 개로 나뉘어 계정과목별 검토나 증빙 대조를 병렬로 수행하는 구조에서, 최종 결과만 남고 각 서브에이전트의 판단 과정이 사라진다면 그 결론을 감사 증거로 채택하기 어렵다. 텍스트·사고 포워딩은 이런 중간 판단 과정을 로그로 남겨 사후 검증 가능하게 만드는 장치다.

내부통제 관점에서도, AI 에이전트가 수행한 절차를 재구성할 수 있는지는 IT 일반통제(ITGC) 실사에서 갈수록 중요한 점검 항목이 되고 있다. 서브에이전트별로 트랜스크립트를 남길 수 있는지, 그 로그가 위변조 없이 보존되는지는 벤더 평가 체크리스트에 들어갈 만한 질문이다.

다만 이 기능이 기본값이 아니라 옵트인이라는 점도 눈여겨볼 만하다. 모든 것을 항상 기록하면 로그량과 민감정보 노출 위험이 커지므로, 어떤 작업에서 이 수준의 가시성이 필요한지 선별하는 판단 자체가 실무에서는 또 다른 설계 과제가 된다.

## 핵심 용어·논쟁

- stream-json — 이벤트 하나당 JSON 한 줄로 실시간 스트리밍하는 Claude Code의 출력 형식(NDJSON).
- `parent_tool_use_id` — 스트림의 각 메시지가 어느 도구 호출(서브에이전트 생성)에서 파생됐는지 식별하는 필드. 메인 대화는 `null`.
- 서브에이전트(Subagent) — 별도의 컨텍스트 창에서 위임받은 작업을 독립적으로 수행하고 요약만 반환하는 Claude Code의 하위 실행 단위.
- 트랜스크립트 재구성 — 개별 이벤트 로그를 시간순·발신자순으로 이어 붙여 전체 대화·사고 흐름을 복원하는 작업.

## 자료 깊이 읽기

### Run Claude Code programmatically — 영어, 공식 문서, 중급
Claude Code 공식 헤드리스 모드 문서로, `-p` 모드의 전체 CLI 옵션과 stream-json 출력 구조를 정리한다. 특히 "Stream responses" 절에서 서브에이전트 메시지가 `assistant`·`user` 타입으로 스트림에 섞여 들어오며 `parent_tool_use_id`로 식별된다는 점, 기본값에서는 `tool_use`·`tool_result`만 노출되고 `--forward-subagent-text`(v2.1.211 이상)를 켜야 텍스트·thinking까지 노출된다는 점을 정확한 필드 표와 함께 설명한다. `jq`를 이용한 실제 필터링 예시 코드도 포함되어 있어 바로 따라 해볼 수 있다.

### 사용자 정의 subagent 만들기 — 한국어, 공식 문서, 초급
Claude Code 한국어 공식 문서로, 서브에이전트가 왜 필요한지(컨텍스트 보존, 도구 제약, 비용 절감)와 Explore·Plan·general-purpose 같은 내장 서브에이전트의 특성을 설명한다. 서브에이전트가 "자신의 컨텍스트 윈도우에서 작업을 수행하고 요약만 반환한다"는 기본 동작 원리를 먼저 이해해야 텍스트 포워딩 기능이 왜 필요한지 납득할 수 있어, 오늘 주제의 사전 지식으로 적합하다.
