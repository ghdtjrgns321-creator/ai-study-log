# 오늘의 AI 개념: 아티팩트 MCP 커넥터 연동(Artifacts MCP Connectors)

> 작성일: 2026-07-24 · 분류: claude-code

## 한 줄 정의

클로드 코드의 아티팩트(공유 가능한 웹페이지 결과물)가 페이지를 열 때마다 외부 데이터를 직접 불러와, 만든 시점의 스냅숏이 아니라 항상 최신 상태를 보여주게 하는 기능이다.

## 쉬운 설명

아티팩트(Artifact)는 클로드 코드가 세션 결과를 터미널 텍스트 대신 브라우저에서 보는 웹페이지로 만들어 공유하는 기능이다. 기존에는 아티팩트를 만든 시점의 데이터만 페이지에 고정되어, 시간이 지나면 내용이 낡아졌다.

이번 업데이트로 아티팩트가 MCP(Model Context Protocol, AI가 외부 도구·데이터에 접속하는 표준 규격) 커넥터를 페이지가 열릴 때마다 다시 호출할 수 있게 됐다. 예를 들어 깃허브 이슈 현황판 아티팩트를 만들면, 페이지를 열 때마다 깃허브 커넥터를 통해 그 순간의 실제 이슈 목록을 가져와 보여준다.

비유하자면 기존 아티팩트가 "그날 찍은 사진"이었다면, MCP 커넥터가 붙은 아티팩트는 "지금 이 순간을 비추는 창문"에 가깝다. 다만 이 창문은 페이지를 보는 사람 각자의 창문이라는 점이 특이하다.

기존 정적 아티팩트와 다른 점은 데이터 출처다. 정적 아티팩트는 세션이 수집해 페이지에 박아 넣은 값을 그대로 보여주지만, 커넥터 연동 아티팩트는 페이지를 보는 사람의 클라우드 계정에 연결된 MCP 커넥터를 매번 호출해 그 사람이 접근 가능한 데이터만 보여준다.

## 동작 원리

1. 사용자가 "깃허브 커넥터로 열린 PR 목록을 불러오는 대시보드를 만들어 달라"는 식으로 어떤 커넥터에서 어떤 데이터를 가져올지 프롬프트에 명시한다.
2. 클로드 코드는 페이지를 게시할 때 이 페이지가 호출할 수 있는 커넥터 목록을 함께 선언한다. 선언되지 않은 커넥터는 페이지가 호출할 수 없다.
3. 다른 사람이 페이지를 열면, claude.ai가 그 사람에게 커넥터 접근 권한을 먼저 승인받는다. 승인을 거부하거나 해당 커넥터를 연결하지 않은 사람은 실시간 영역이 빈 채로 페이지만 본다.
4. 승인 후에는 페이지가 아니라 claude.ai가 뷰어 계정의 연결 정보로 실제 호출을 대신 수행한다. 페이지 자체는 누구의 자격 증명도 보지 못한다.
5. 응답은 뷰어의 브라우저에 캐시되어, 재방문 시 캐시를 먼저 보여준 뒤 최신 결과로 갱신한다.

## 구체 예시·사례

깃허브 미결 PR 현황판 아티팩트를 팀에 공유했다고 가정한다. 이 페이지를 여는 사람이 각자 다른 깃허브 계정으로 연결돼 있다면, 같은 링크를 열어도 A가 보는 PR 목록과 B가 보는 PR 목록이 서로 다를 수 있다. 각자 자기 계정이 접근 가능한 저장소의 데이터만 보이기 때문이다.

이 특성 때문에 커넥터가 연동된 아티팩트는 공개 링크로 공유할 수 없다. Pro·Max 요금제에서는 공개 링크가 유일한 공유 수단이라, 커넥터 연동 페이지는 본인에게만 비공개로 남는다. Team·Enterprise 요금제에서는 조직 내부 공유까지만 가능하다.

## 비슷한 것과 비교

| 구분 | 정적 아티팩트 | MCP 커넥터 연동 아티팩트 | 전통 BI 대시보드 |
|------|------|------|------|
| 데이터 시점 | 생성 시점 스냅숏 | 열람 시점 실시간 | 구축된 파이프라인 기준 실시간·배치 |
| 자격 증명 | 불필요(고정 데이터) | 뷰어 각자의 계정 사용 | 별도 로그인·서비스 계정 필요 |
| 구축 비용 | 세션 한 번으로 완성 | 세션 한 번 + 커넥터 연결 | 데이터 엔지니어링·백엔드 구축 필요 |
| 공개 공유 | 가능 | 불가(비공개·조직 내부만) | 도구별 상이 |

일회성 결과 공유나 공개 배포에는 정적 아티팩트를, 팀 내부에서 항상 최신 값을 보여줘야 하는 소규모 현황판에는 커넥터 연동 아티팩트를, 대규모·다부서용 정식 대시보드에는 전통 BI 도구를 선택하는 편이 합리적이다.

## 왜 지금 중요한가

이 기능은 클로드 코드 2026년 7월 13~17일자(Week 29, 버전 v2.1.207~v2.1.212) 업데이트로 공식 발표됐다. Pro·Max·Team·Enterprise 요금제에서 클로드 코드 v2.1.209 이상에 적용되며, 공개 공유 아티팩트에는 적용되지 않는다.

세션이 끝난 뒤에도 데이터가 계속 갱신된다는 점에서, 일회성 보고서 생성 도구였던 아티팩트가 가벼운 실시간 업무 도구에 가까워지고 있다는 평가가 나온다.

## 회계법인 AI 직무 연결 포인트

감사·재무 실무에서는 진행 중인 감사 항목 현황, 미해결 질의 목록, 이상거래 탐지 결과 등을 팀원이 항상 최신 상태로 확인해야 하는 경우가 많다. 커넥터 연동 아티팩트를 쓰면 별도 대시보드 시스템을 구축하지 않고도, 사내 시스템에 연결된 MCP 커넥터를 통해 감사 진행 현황판을 가볍게 공유할 수 있다.

다만 각 뷰어가 자기 계정 권한으로만 데이터를 보는 구조이므로, 감사 정보처럼 접근 권한 통제가 중요한 데이터를 다룰 때는 커넥터 자체의 권한 설계(누가 어떤 데이터에 접근 가능한지)가 여전히 핵심 통제 지점으로 남는다.

면접에서는 "생성형 AI 산출물을 어떻게 최신 상태로 유지하며 공유할 것인가"라는 질문에, 세션 스냅숏과 실시간 커넥터 호출의 차이, 그리고 뷰어별 권한 분리 구조를 근거로 답할 수 있다.

## 핵심 용어·논쟁

- **MCP(Model Context Protocol)** — AI가 외부 도구·데이터 소스에 표준화된 방식으로 접속하도록 정의한 프로토콜.
- **커넥터(Connector)** — claude.ai 계정에 연결된 외부 서비스(깃허브, 슬랙 등) 접속 통로.
- **CSP(Content Security Policy)** — 아티팩트 페이지가 외부로 임의 요청을 보내지 못하도록 막는 브라우저 보안 정책. 커넥터 호출만 예외로 허용된다.
- **뷰어별 자격 증명 분리** — 페이지 제작자가 아니라 각 열람자의 계정 권한으로 데이터를 조회하는 구조.

## 자료 깊이 읽기

### Share session output as artifacts — Claude Code 공식 문서 — 영어, 문서, 중급
클로드 코드 공식 문서의 아티팩트 전체 가이드다. 아티팩트를 만들고 공유하고 갱신하는 기본 흐름에 이어, "Pull live data with MCP connectors" 절에서 커넥터 연동의 정확한 동작 방식을 설명한다. 페이지가 게시될 때 호출 가능한 커넥터를 선언하는 절차, 뷰어가 처음 열람할 때 접근 권한을 승인하는 절차, 호출이 페이지가 아니라 claude.ai를 통해 뷰어 계정으로 이뤄진다는 점, 커넥터가 연동된 페이지는 어떤 요금제에서도 공개 링크로 공유할 수 없다는 제약을 구체적으로 명시한다. 페이지 크기 16MiB 제한, 백엔드 없는 정적 페이지라는 구조적 한계도 함께 다뤄, 아티팩트를 무엇에 쓸 수 있고 없는지 판단하는 기준을 제공한다.

### Week 29 · July 13–17, 2026 — Claude Code Docs — 영어, 문서, 초중급
클로드 코드의 주간 업데이트 다이제스트로, 이번 기능을 "Artifacts call your MCP connectors"라는 제목으로 소개한다. 대시보드가 세션 당시의 스냅숏이 아니라 조회 시점의 실시간 데이터를 보여준다는 핵심 변화를 한 문단으로 요약하고, 프롬프트 예시("깃허브 커넥터로 열린 PR 목록을 불러오는 대시보드 아티팩트를 만들어 달라")를 제시한다. 같은 주에 함께 나온 화면 낭독기 모드, `/fork` 백그라운드 세션 전환, WebSearch·서브에이전트 세션 호출 한도(기본 200회) 등 다른 변경 사항도 함께 정리돼 있어, 이번 주 업데이트 전체 맥락을 파악하는 데 유용하다.
