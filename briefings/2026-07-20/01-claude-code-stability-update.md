# 오늘의 AI 개념: 클로드 코드 안정성·안전성 업데이트(Claude Code Stability & Safety Update, EndConversation·진행 하트비트)

> 작성일: 2026-07-20 · 분류: claude-code

## 한 줄 정의

클로드 코드에 학대·탈옥 시도가 계속될 때 스스로 세션을 끊는 기능과, 오래 걸리는 작업이 멈춘 게 아니라 진행 중이라는 신호를 주기적으로 보여주는 기능이 함께 추가된 안정성·안전성 업데이트다.

## 쉬운 설명

2026년 7월 18일 배포된 클로드 코드 v2.1.214는 EndConversation 툴, 진행 하트비트(Progress Heartbeat), 여러 권한검사 결함 수정을 한데 묶은 릴리스다. EndConversation은 클로드가 대화 자체를 끝낼 수 있게 하는 도구이고, 하트비트는 도구 호출(tool call) 중 살아있음을 알리는 짧은 상태 표시를 뜻한다.

비유하자면 콜센터 상담원이 반복적인 폭언을 들은 뒤 정중한 경고를 거쳐 통화를 끊을 권한을 갖게 된 것과 비슷하다. 진행 하트비트는 택배 조회 화면에서 "배송 중"이라는 상태가 계속 갱신되는 것과 같다. 화면이 멈춘 것처럼 보여도 실제로는 작업이 진행되고 있다는 사실을 알려준다.

이 기능은 완전히 새로운 개념은 아니다. 앤트로픽은 이미 2025년 8월부터 claude.ai의 Opus 4·4.1 모델에 같은 취지의 기능을 넣은 바 있다. 다만 그때는 "모델 복지(model welfare)"라는 관점에서 챗봇 대화창을 대상으로 했고, 이번에는 파일·코드베이스에 직접 접근하는 CLI(명령줄 인터페이스) 세션을 대상으로 범위가 넓어졌다는 점이 다르다.

## 동작 원리

EndConversation 툴은 다음 절차를 거쳐야 실행된다.

1. 사용자의 지속적인 학대나 탈옥(jailbreak) 시도가 감지되면, 클로드는 먼저 여러 차례 대화 방향을 돌리려 시도한다.
2. 그래도 상황이 이어지면 "계속하면 세션을 종료하겠다"는 명시적 경고를 한 번 보낸다.
3. 경고 이후에도 상황이 반복되거나, 사용자가 데모 목적으로 직접 종료를 요청하면 도구를 실행한다.
4. 도구가 실행되면 해당 세션에서는 클로드가 더 이상 응답하지 않는다. 작업을 이어가려면 새 세션을 시작해야 한다.
5. 자해·정신적 위기·즉각적 폭력 등이 의심되는 상황에서는 사용자의 행동과 무관하게 이 도구를 절대 사용하지 않는다.
6. 백그라운드 서브에이전트(포크) 안에서는 이 도구가 아무 동작도 하지 않는 무효(no-op) 처리로 설계되어 있어, 메인 세션을 실수로 종료시킬 수 없다.

진행 하트비트는 이전까지 응답 없이 조용했던 장시간 도구 호출 중에 주기적으로 상태를 알리는 방식으로 동작한다. 나흘 앞선 v2.1.210(7월 14일)에서 먼저 경과 시간 카운터를 도입해 도구 호출 줄이 멈추지 않고 흐르는 것처럼 보이게 했고, v2.1.214에서 이를 하트비트 형태로 확장했다. 다만 정확한 발신 주기나 메시지 형식은 공식 문서에 구체적으로 공개되어 있지 않다.

같은 릴리스에는 권한검사 관련 수정도 다수 포함됐다. `Edit(src/**)`처럼 한 단계짜리 허용 규칙이 트리 어디에서나 같은 이름의 폴더에 잘못 적용되던 결함, 윈도우 PowerShell 5.1 세션에서의 권한검사 우회, 1만 자를 넘는 명령이 자동 승인되던 문제, `pkill -f`가 클로드 코드 자신의 프로세스를 죽일 수 있던 결함 등이 함께 고쳐졌다.

## 구체 예시·사례

이전에는 여러 분 걸리는 배치 작업(예: 대량 문서 검토 스크립트)을 실행하면 화면에 아무 표시가 없어 멈춘 것인지 실행 중인지 구분하기 어려웠고, 사용자가 이를 오작동으로 오해해 세션을 강제 종료하는 일이 잦았다. v2.1.214 이후에는 이런 도구 호출 중에도 주기적으로 상태가 갱신되어 작업이 살아있음을 확인할 수 있다.

또 다른 예로, 누군가 클로드 코드에 안전 지침을 무시하도록 유도하는 탈옥 시도를 반복하거나 여러 차례 경고에도 폭언을 멈추지 않는 경우, 클로드는 EndConversation을 실행해 해당 세션을 종료한다. 이후에는 같은 세션에서 대화를 이어갈 수 없고 새 세션을 열어야 한다.

## 비슷한 것과 비교

| 구분 | claude.ai (2025년 8월~) | 클로드 코드 (2026년 7월~) |
|------|------|------|
| 대상 | 챗봇 대화창 | 파일·코드베이스에 접근하는 CLI 세션 |
| 도입 배경 | 모델 복지(model welfare) 관점 | 동일한 정책을 CLI로 확장 |
| 종료 단위 | 해당 채팅 스레드 | 해당 터미널 세션 전체 |
| 백그라운드 처리 | 해당 없음 | 서브에이전트(포크)에서는 무효 처리 |
| 부가 기능 | 없음 | 권한검사·Bash·PowerShell 수정과 함께 배포 |

두 경우 모두 최후 수단으로만 쓰이고, 명시적 경고를 먼저 거치며, 위기 상황에서는 절대 사용하지 않는다는 정책 논리는 동일하다. 다만 클로드 코드는 파일 접근 권한이 걸린 세션을 종료하는 것이므로, 챗봇 대화 종료보다 실무 영향이 크다.

## 왜 지금 중요한가

이 업데이트는 클로드 코드 v2.1.214로 2026년 7월 18일 배포됐다는 사실이 공식 체인지로그에 명시돼 있다.

같은 시기 클로드 코드·클로드 기반 도구를 둘러싼 보안 이슈가 여러 건 보도됐다. 예를 들어 셸 명령을 50개 이상 하위 명령으로 연결하면 거부(deny) 규칙이 적용되지 않는 결함, 클로드용 브라우저 확장의 취약점, 깃허브 액션 연동 결함 등이다. 다만 공식 체인지로그는 이런 사건과 v2.1.214를 직접 연결 짓지는 않으므로, 이는 업계 전반의 압력이라는 배경 정도로 이해하는 것이 정확하다.

## 회계법인 AI 직무 연결 포인트

감사 업무에서 클로드 코드로 대량 원장 대조나 문서 일괄 검토 같은 장시간 배치 작업을 돌릴 때, 진행 하트비트는 그 작업이 멈춘 게 아니라 진행 중이라는 사실을 확인할 수 있게 해준다. 민감한 재무 데이터를 다루는 다분 단위 자동화 작업에서 이런 생존 신호는 신뢰도를 높이는 요소다.

이번에 강화된 권한검사(1만 자 초과 명령 자동 확인, 원격 세션에서의 승인 절차 수정 등)는 에이전트에게 재무 데이터 파일에 대한 읽기·쓰기 권한을 부여할 때, 의도한 범위를 벗어난 실행을 막아주는 통제 장치 역할을 한다.

EndConversation 자체는 일상적인 감사 업무 기능은 아니지만, 민감한 데이터를 다루는 에이전트에 대해 앤트로픽이 적대적 사용·탈옥 시도에 대응하는 방어선을 CLI까지 확장하고 있다는 신호로 볼 수 있다.

## 핵심 용어·논쟁

- **EndConversation 툴** — 지속적인 학대·탈옥 시도 시, 여러 차례 유도와 명시적 경고 후에만 클로드가 스스로 세션을 종료하는 도구.
- **진행 하트비트(Progress Heartbeat)** — 예전에는 응답 없이 조용했던 장시간 도구 호출 중 주기적으로 "살아있음"을 알리는 상태 표시.
- **권한검사(Permission Check)** — 클로드 코드가 Bash·PowerShell·Edit 등 도구 실행 전 허용·확인·거부 규칙에 따라 자동 승인 여부를 판단하는 내부 로직.
- **우회(Bypass)** — 의도한 보안 규칙(예: 거부 규칙)이 특정 입력 패턴에서 실제로는 적용되지 않고 넘어가는 결함.
- **원격 측정(Telemetry)** — 세션·메시지·도구 호출 단위로 로그를 남겨 문제를 추적할 수 있게 하는 계측 데이터.

EndConversation은 "사용자 보호"가 아니라 "모델 복지" 관점에서 설계됐다는 점이 논쟁의 대상이다. AI 모델에게 대화를 끊을 권한을 주는 것이 실제로 모델을 보호하는 조치인지, 아니면 마케팅적 프레이밍에 가까운지에 대해 업계에서 의견이 갈린다.

## 자료 깊이 읽기

### Claude Code changelog — 영어, 공식 문서, 쉬움
클로드 코드의 버전별 변경 사항을 모아둔 공식 페이지다. v2.1.214(2026년 7월 18일) 항목에서 EndConversation 툴 추가와 진행 하트비트 도입을 명시하고, claude.ai에서는 2025년부터 있던 기능이라고 밝힌다. 같은 버전에 `Edit(src/**)` 허용 규칙 오류, 윈도우 PowerShell 5.1 권한검사 우회, 1만 자 초과 명령 자동 승인 문제 등 다수의 권한·보안 수정이 함께 실렸다. 하루 뒤인 v2.1.215에서는 `/verify`·`/code-review` 스킬을 클로드가 스스로 실행하지 않고 사용자가 직접 호출해야 하는 것으로 바뀌었다는 내용도 확인된다. 이번 검색에서 실제로 접속해 본문을 확인했다.

### Piebald-AI/claude-code-system-prompts (GitHub) — 영어, 소스코드 저장소, 중상급
클로드 코드 바이너리에서 실제 시스템 프롬프트 문자열을 추출해 공개하는 저장소다. `tool-description-endconversation.md` 파일에는 EndConversation을 언제 어떻게 써야 하는지에 대한 실제 지침 원문이 담겨 있다. 클로드 자신의 좌절이나 요청 거부만으로는 쓸 수 없고, 지속적이고 진짜인 학대 상황에서만, 그것도 여러 차례 방향 전환과 명시적 경고를 거친 뒤에만 쓸 수 있다고 명시한다. 자해·정신적 위기·즉각적 폭력이 의심되면 사용자의 행동과 무관하게 이 도구를 절대 고려하지 않는다는 예외 조항도 담겨 있다. 백그라운드 포크에서는 이 도구가 무효 처리된다는 별도 시스템 리마인더 파일도 함께 확인된다. 이번 검색에서 실제로 접속해 본문을 확인했다.
