# 영상 정리: Claude Code 왕초보 입문 튜토리얼 23가지 팁

> 원본: [Claude Code 왕초보 입문 튜토리얼 23가지 팁 | 클로드코드 설치 사용법 50분 강의 완전정복 2026 가이드](https://www.youtube.com/watch?v=1_bRmkUvjHA) — 한국어, 유튜브, 약 50분, 채널 "AI 싱커클럽"
> 연결된 브리핑: [01-claude-code-agent-skills-plugin-ecosystem.md](../01-claude-code-agent-skills-plugin-ecosystem.md)
> 이 문서는 자동 생성 한국어 자막(음성 인식 기반)을 직접 내려받아 확인하고 정리한 것이다. 자막 인식 오류로 일부 단어가 부정확할 수 있으나, 내용 흐름과 팁의 실제 순서·명령어는 원 영상과 대조해 정리했다.

## 영상의 구성

영상은 "23가지 팁"을 난이도별 3레벨로 나눠 설명한다.

- **레벨 1 (기초, 팁 1~9)** — 설치부터 첫 프로젝트까지. 처음 쓰는 사람도 바로 따라 할 수 있는 수준.
- **레벨 2 (중급, 팁 10~17)** — 자동 승인, 플랜 모드, 사고(think) 모드 등 작업 속도를 실질적으로 높여주는 기능.
- **레벨 3 (고급 맛보기, 팁 18~23)** — 워크트리 병렬 작업, GitHub 연동 등 익숙해진 뒤에 쓰는 기능. 영상에서는 "맛보기" 수준으로만 다룬다.

강사는 회계·개발 무경험자도 따라 할 수 있게 터미널 개념부터 설명하는데, 이 요약에서는 초심자용 설명은 걷어내고 **팁의 핵심과 명령어**만 순서대로 정리한다.

## 레벨 1 — 기초 (설치부터 첫 결과물까지)

1. **설치**: `npm install -g @anthropic/claude-code` (영상 기준 표기, 공식 문서 명령어 그대로 사용 권장). 설치 중 권한 에러가 나면 관리자 권한(sudo)으로 재시도.
2. **최초 실행과 로그인**: 터미널에서 `claude` 입력 → 작업 폴더 신뢰 여부 확인 → Claude 계정으로 로그인(구독 계정 사용, Console API 계정은 별도 과금이라 초심자에게는 비추천) → 터미널에 인증 코드를 붙여넣어 완료.
3. **첫 프로젝트 실행**: 새 폴더에서 `claude` 실행 후 자연어로 "간단한 계산기 웹사이트 만들어줘"처럼 요청하면 HTML/CSS/JS를 즉시 생성하고 실행 결과까지 확인 가능. 이것이 이른바 "바이브 코딩"의 최소 사례다.
4. **세션 이어가기**: `/exit`으로 나가도 다음에 `claude --continue`(또는 `claude -c`)로 직전 세션을 이어서 진행할 수 있다. Claude Code는 이전 대화 맥락을 기억한다.
5. **투두리스트 자동 생성**: 복잡한 작업을 시키면 Claude가 스스로 할 일 목록(To-do)을 세우고 하나씩 처리하며 진행 상황을 보여준다. 단순 작업에서는 생략되기도 한다.
6. **CLAUDE.md — 프로젝트 총괄 지침 파일**: 영상에서 "이거 하나만 제대로 해도 품질이 두 배 달라진다"고 강조하는 핵심 팁. `/init` 명령으로 현재 코드베이스를 분석해 기본 틀을 자동 생성한 뒤, 그 초안을 그대로 쓰지 말고 프로젝트에 맞게 직접 다듬어야 한다. 예: "커밋 메시지는 한글로 작성", "모든 답변과 설명은 한글로 한다" 같은 규칙을 넣으면 세션이 바뀌어도 매번 같은 지시를 반복할 필요가 없다. Anthropic 공식 관계자도 "Claude Code의 진짜 힘은 CLAUDE.md에서 나온다"고 언급했다고 소개된다.
7. **ESC로 중단·롤백**: 작업 도중 ESC 한 번 = 현재 작업 중단, ESC 두 번 = 이전 상태로 롤백(대화 되감기). 원치 않는 방향으로 진행될 때 유용하다.
8. **`/doctor` 명령**: Claude Code 자체의 설치 상태·버전·경로 등을 진단해주는 헬스체크 명령. 문제가 의심될 때 가장 먼저 실행.
9. **이미지 드래그 앤 드롭**: 에러 화면 캡처나 원하는 디자인 스크린샷을 터미널 창에 그대로 드래그해 넣으면 Claude가 이미지를 이해하고 그에 맞춰 디버깅하거나 디자인을 반영한다.

여기에 더해 비용 관련 팁도 짧게 언급된다: 요금제는 Pro(월 20달러) / Max(월 100달러 또는 200달러) 구조이며, 강사는 실사용량 기준으로 Max 200달러가 오히려 "가성비"라고 평가한다(Pro 대비 20배 한도, 가격은 10배).

## 레벨 2 — 중급 (작업 속도를 바꾸는 기능)

10. **자동 승인 모드(Shift+Tab)**: 매번 "이거 해도 될까요?"라고 묻는 승인 과정을 건너뛰는 모드. Shift+Tab을 눌러 일반 모드 → 플랜 모드 → 자동 승인 모드를 순환 전환한다. 완전히 신뢰할 때만 사용 권장.
11. **`--dangerously-skip-permissions` (혹은 세션 내 `skip permissions` 입력)**: 모든 권한 확인을 건너뛰고 전권을 위임하는 모드. 강사는 이를 개인적으로 "클로드 고(Claude Go)"라고 부르며, 손에 익은 뒤에만 권장한다.
12. **플랜 모드**: Anthropic 대표가 "작업 시간의 90%를 이 모드로 쓴다"고 언급했다는 핵심 기능. 코드를 바로 짜지 않고 먼저 프로젝트 구조를 파악하고 계획을 세운 뒤, 사용자가 승인해야만 실제 구현에 들어간다. 복잡한 작업일수록 플랜 모드를 먼저 거치는 것이 실패율을 크게 낮춘다.
13. **사고(Thinking) 모드**: 복잡한 문제에서 Claude가 내부적으로 먼저 깊게 추론하는 기능. 2026년 1월부터는 기본적으로 켜져 있으며(기본 최대 사고 토큰 약 3만), 과거에 쓰던 `ultrathink`/`think hard` 같은 매직 키워드는 더 이상 효과가 없다(1월 16일 업데이트로 폐지). 끄고 싶으면 Mac은 Option+T, Windows는 Alt+T. 파워 유저는 환경 변수로 사고 토큰 한도를 최대 6만까지 확장 가능.
14. **`@` 파일 참조**: `@파일명`으로 특정 파일만 콕 집어 지정해 그 파일만 수정하게 할 수 있다. 리팩터링이나 의존성 업데이트처럼 범위를 좁히고 싶을 때 유용.
15. **모델 변경 (`/model`)**: Claude Code는 기본적으로 가장 똑똑하고 비싼 모델(Opus 계열)을 쓴다. 간단한 작업엔 과한 선택일 수 있어, 균형 잡힌 작업엔 Sonnet, 빠르고 저렴한 단순 검색·파일 탐색엔 Haiku로 낮춰 비용을 관리할 수 있다. 복잡한 설계·디버깅에는 Sonnet/Opus를 권장.
16. **`/compact`**: 길어진 대화 내용을 요약해서 압축 저장하는 명령. `/clear`(전체 삭제)와 달리 이전 맥락을 요약본으로 남겨두기 때문에 토큰을 절약하면서도 맥락 손실을 최소화한다. 최근 버전은 대화가 너무 길어지면 자동으로 셀프 컴팩팅을 수행한다. `/context` 명령으로 현재 토큰 사용량을 시각적으로 확인할 수 있다.
17. **웹 검색·문서·PDF 읽기**: Claude Code는 인터넷 검색이 가능해 라이브러리 공식 문서를 스스로 찾아 참고한다. PDF 파일도 드래그 앤 드롭으로 넣으면 텍스트를 추출하고 요약해준다.

## 레벨 3 — 고급 맛보기 (익숙해진 뒤 쓰는 기능)

18. **인터뷰 기반 요구사항 정의**: 그냥 "이거 만들어줘"라고 하는 대신, "만들기 전에 나한테 먼저 인터뷰해줘. 기술 구현·UI/UX·트레이드오프를 상세히 질문하고, 끝나면 계획을 파일로 저장해줘"라고 요청하는 기법. 객관식 선택지 개수·전체 질문 수를 미리 지정하면(예: "객관식 4개 + 주관식 1개, 질문은 총 5개") 답변 부담을 줄이면서도 체계적인 스펙 문서를 뽑아낼 수 있다. 강사는 이를 "정말 고급 팁"이라고 강조하며, 막연히 시키는 것과는 결과 품질이 크게 다르다고 말한다.
19. **핸드오프(Handoff) 문서화**: `/compact`가 대화를 요약만 한다면, "지금까지 작업 내용을 handoff.md로 정리해줘(시도한 것·성공한 것·실패한 것 포함)"라고 요청하면 별도 인수인계 문서가 만들어진다. `/clear`로 세션을 완전히 비운 뒤에도 이 문서를 참조시키면 다음 세션이 이전 맥락을 그대로 이어받을 수 있다.
20. **커스텀 명령/스킬 만들기**: "체인지로그를 자동으로 기록하는 `/changelog` 명령을 만들어줘"처럼 요청하면 나만의 반복 작업 자동화 명령을 만들 수 있다(최근 버전에서는 커스텀 커맨드 대신 스킬로 저장되는 경우도 있다고 언급).
21. **Git 워크트리를 이용한 병렬 작업**: 기능별로 별도 브랜치 폴더(worktree)를 만들어 터미널을 여러 개 띄우고 각각 다른 기능(A, B, C)을 동시에 개발하게 하는 기법. 한 뿌리 저장소에서 여러 작업을 병렬로 진행한 뒤 나중에 병합한다. 강사는 "이걸 제대로 쓰면 일주일 작업을 몇 시간으로 끝낼 수 있다"고 언급하지만, 초심자에게는 어렵다고 조언한다.
22. **GitHub 앱 연동**: `설치 GitHub 앱`을 실행해 인증하면, 터미널을 열지 않고도 GitHub 이슈에 "버그 수정해줘"라고 남기는 것만으로 Claude가 코드를 수정하고 PR까지 만들어 올린다.
23. **명령어 즉시 실행 접두사(`!`)**: 예를 들어 `git status`를 그냥 입력하면 Claude가 이를 해석하느라 시간이 걸리는데, 앞에 느낌표를 붙여 `!git status`처럼 입력하면 해석 과정 없이 즉시 실행되어 훨씬 빠르다. 그리고 작업 중에 다음 지시를 미리 입력해두면 대기열(큐)에 쌓여 현재 작업이 끝나는 대로 순차 실행된다(ESC로 대기열 취소 가능).

## 핵심 요약 (강사가 강조한 3단계)

1. **레벨 1**: 설치 → `/init` → CLAUDE.md로 규칙 고정.
2. **레벨 2**: 플랜 모드가 핵심, Thinking 모드로 깊은 추론을 유도.
3. **레벨 3**: 익숙해지면 워크트리로 병렬 작업까지 확장.

영상은 "레벨 1·2만 익혀도 바로 실무에 쓸 수 있다"고 강조하며, 후속 영상에서 서브에이전트·MCP·훅(Hook)·스킬 등 확장 기능을 다룰 예정이라고 예고한다.

## 이 영상에서 주의할 점

- 자동 생성 자막 기반 정리라 일부 세부 워딩(예: 정확한 CLI 플래그명)은 영상을 직접 보며 재확인하는 것이 안전하다. 특히 `--dangerously-skip-permissions` 같은 위험도 높은 옵션은 실제 공식 문서로 정확한 플래그명과 부작용을 재확인한 뒤 사용할 것.
- "Thinking 모드가 기본 켜짐"이라거나 "ultrathink 키워드 폐지" 같은 버전 종속적 사실은 영상 녹화 시점(2026년 1월 기준 언급) 기준이므로, 현재 Claude Code 버전의 changelog로 최신 여부를 교차 확인해야 한다.
