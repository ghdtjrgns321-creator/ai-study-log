# 오늘의 AI 개념: 서브에이전트 세션 예산 상한(Max Subagents Per Session)과 백그라운드 에이전트 자율 커밋·PR

> 작성일: 2026-07-23 · 분류: claude-code

## 한 줄 정의

한 세션이 무한히 새 서브에이전트를 만들어 폭주하지 못하도록 세션당 생성 개수에 상한을 두고, 그렇게 통제된 서브에이전트가 백그라운드에서 스스로 커밋·푸시·초안 PR까지 만들도록 한 클로드 코드(Claude Code)의 실행 통제 장치다.

## 쉬운 설명

서브에이전트(sub-agent)는 메인 에이전트가 일을 쪼개 맡기는 하위 작업자다. 그런데 하위 작업자가 또 하위 작업자를 부르고, 그게 다시 부르는 식으로 번지면 한 세션이 수백 개의 에이전트를 찍어내며 구독 사용량과 토큰을 순식간에 태운다. 세션 예산 상한은 여기에 "이번 세션에서 서브에이전트는 최대 몇 개까지"라는 총량 브레이크를 건 것이다.

비유하자면 한 팀장이 프로젝트마다 계약직을 무제한으로 뽑을 수 있으면 인건비가 폭발한다. 회사가 "프로젝트당 채용 200명까지, 프로젝트가 끝나면(=`/clear`) 카운터 리셋"이라는 헤드카운트 정책을 박아두는 것과 같다.

두 번째 축은 자율성이다. 예전에는 서브에이전트가 코드를 고쳐도 사람이 직접 커밋·푸시·PR을 만들어야 했다. 이제 백그라운드 서브에이전트는 작업이 끝나면 스스로 커밋하고 푸시하고 **초안 PR(draft pull request)**까지 열어둔다. 통제(상한)와 위임(자율 실행)을 동시에 강화해, "마음껏 풀어주되 폭주는 막는다"는 방향이다.

## 동작 원리

1. **카운터 초기화** — 세션이 시작되면 서브에이전트 생성 카운터가 0에서 출발한다. 상한은 환경변수 `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`로 지정하며 기본값은 200이다.
2. **생성 시마다 증가** — 메인 에이전트나 스킬이 서브에이전트를 띄울 때마다 카운터가 1씩 오른다. 중첩(서브에이전트가 또 서브에이전트를 부름)도 같은 세션 예산에서 차감된다.
3. **상한 도달 시 차단** — 카운터가 상한에 닿으면 추가 생성 요청이 막힌다. 폭주 루프의 안전판이다.
4. **리셋** — `/clear`로 컨텍스트를 비우면 이 예산도 함께 리셋된다.
5. **백그라운드 실행 + 자율 마무리** — `context: fork`로 도는 스킬은 기본적으로 백그라운드에서 실행된다(스킬별로 `background: false`로 끌 수 있음). 백그라운드에서 코드를 편집한 에이전트는 완료 시 커밋·푸시하고 초안 PR을 열며, 이 행위는 `claude agents` 뷰에 연결된 참조로 표시된다.

## 구체 예시·사례

리팩터링 작업에서 메인 에이전트가 파일 30개를 30개 서브에이전트에 병렬로 맡긴다고 하자. 각 서브에이전트가 "관련 파일도 손봐야겠다"며 추가로 3~4개씩 더 띄우면 순식간에 100개를 넘긴다. 여기에 검증 에이전트까지 붙으면 200개 근처. 기본 상한 200에 걸리면 그 이상은 생성이 거부되어, 실수로 짜인 재귀 위임이 사용량을 무한정 태우는 사고가 차단된다. 대규모 마이그레이션처럼 정당하게 200개 이상이 필요하면 `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=500`처럼 명시적으로 올린다 — "명시적으로 올렸다"는 사실 자체가 폭주가 아니라 의도임을 드러낸다.

## 비슷한 것과 비교

| 항목          | 세션 예산 상한                  | 중첩 깊이 제한(depth 5) | 백그라운드 전환(MCP 2분)     |
| ------------- | ------------------------------- | ----------------------- | ---------------------------- |
| 무엇을 막나   | 세션 전체 서브에이전트 **총량** | 위임 사슬의 **깊이**    | 긴 도구 호출의 **세션 점유** |
| 축            | 개수(가로 폭주)                 | 단계(세로 폭주)         | 시간(블로킹)                 |
| 리셋·해제     | `/clear` 또는 env로 상향        | 구조적 하드 제한        | 임계값 조정·비활성화         |
| 언제 의식하나 | 대량 병렬 팬아웃 시             | 재귀적 위임 설계 시     | 오래 걸리는 MCP 작업 시      |

선택 기준: 폭이 문제면 세션 예산, 사슬 길이가 문제면 깊이 제한 — 둘은 서로를 대체하지 않고 함께 작동한다.

## 왜 지금 중요한가

- 세션당 서브에이전트 상한(기본 200, `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`)은 v2.1.212(2026-07-17)에 도입됐다. [Claude Code Changelog](https://releasebot.io/updates/anthropic/claude-code)
- 백그라운드 에이전트가 커밋·푸시·초안 PR을 여는 기능은 v2.1.208(2026-07-14), `context: fork` 스킬의 기본 백그라운드 실행은 v2.1.217(2026-07-21)에 반영됐다. [Claude Code Changelog(gradually)](https://www.gradually.ai/en/changelogs/claude-code/)
- 멀티에이전트 팬아웃이 실무에 퍼지면서 "위임은 늘리되 비용·폭주는 통제"가 핵심 과제가 됐고, 이번 변경은 그 두 요구를 같은 릴리스 흐름에서 맞춘 것이다.

## 회계법인 AI 직무 연결 포인트

감사에서 서브에이전트 팬아웃은 계정과목별·자회사별로 표본을 병렬 검토하거나, 수천 건의 전표를 나눠 이상징후를 훑는 데 쓰인다. 이때 세션 예산 상한은 감사 자동화의 **비용·리스크 통제**와 직결된다. 한 감사 세션이 몇 개의 하위 에이전트를 쓸 수 있는지 상한을 두면, 예측 못 한 재귀 위임으로 클라우드 비용이 폭증하거나 감사 증거 수집이 통제 불능이 되는 상황을 막는다.

자율 커밋·초안 PR은 감사 조서(working paper)나 검토 코멘트를 자동 반영하는 파이프라인에 유용하나, **감사인의 검토 통제**와 충돌할 수 있다. 초안 PR로 남겨 반드시 사람이 승인하도록 하는 설계는, 자동화 산출물을 "검토 대기" 상태로 붙잡아 두는 감사 통제 관점과 잘 맞는다. 자동 실행의 편익과 4-eyes(이중검토) 원칙 사이의 균형점을 초안 PR이 제도적으로 표현한다.

## 핵심 용어·논쟁

- **서브에이전트(sub-agent)** — 메인 에이전트가 하위 작업을 위임하는 별도 컨텍스트의 작업자.
- **`context: fork`** — 스킬을 현재 컨텍스트에서 분기해 독립 실행시키는 설정. 기본 백그라운드로 동작.
- **초안 PR(draft pull request)** — 병합 불가 상태로 열려 리뷰만 받는 PR. 자동 산출물의 "승인 대기함" 역할.
- **세션 예산(session budget)** — 한 세션에서 소모 가능한 서브에이전트 생성 총량.
- 논쟁: 자율 커밋·PR의 편의와 "사람 검토 전 코드가 원격에 올라가는" 통제 리스크 사이의 트레이드오프. 초안 PR·백그라운드 격리가 완충하지만, 조직별 승인 게이트 설계가 여전히 필요하다.
