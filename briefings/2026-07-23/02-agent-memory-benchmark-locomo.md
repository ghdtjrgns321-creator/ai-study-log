# 오늘의 AI 개념: 에이전트 메모리 벤치마크(LoCoMo)와 토큰 효율 메모리(Token-Efficient Memory)

> 작성일: 2026-07-23 · 분류: ai-concept

## 한 줄 정의

AI 에이전트가 과거 대화를 얼마나 정확히 기억하는지를 표준 문제셋으로 채점하는 벤치마크(LoCoMo)와, 그 정확도를 유지하면서 질의당 토큰을 크게 줄이는 메모리 설계 방식이다.

## 쉬운 설명

에이전트 메모리(agent memory)는 모델이 이전 세션·대화에서 있었던 사실을 다음 대화에서 다시 꺼내 쓰는 능력이다. 문제는 "잘 기억한다"를 어떻게 증명하느냐다. 그래서 등장한 것이 LoCoMo(Long Conversational Memory) 같은 벤치마크다. 여러 세션에 걸친 긴 대화를 주고, "그때 사용자가 말한 그 정보가 뭐였지?"를 묻는 1,540개의 질문으로 정답률을 잰다.

두 번째 축은 비용이다. 가장 단순한 메모리 전략은 지난 대화를 통째로 컨텍스트에 밀어 넣는 것(full-context)이지만, 대화가 길어질수록 질의마다 수만 토큰을 태운다. 토큰 효율 메모리는 "필요한 사실만 뽑아 저장하고, 물어볼 때 관련된 것만 검색해 넣는" 방식으로 같은 정답률을 훨씬 적은 토큰으로 달성한다.

비유하면, 회의 내용을 매번 녹취록 전체를 다시 읽어(full-context) 답하는 사람과, 요점만 카드로 정리해두고 질문에 맞는 카드 몇 장만 꺼내 보는 사람의 차이다. 후자가 더 싸고, 잘 설계하면 정확도까지 앞선다.

## 동작 원리

토큰 효율 메모리(Mem0의 2026년 4월 알고리즘 기준)의 두 축은 다음과 같다.

1. **단일 패스 추출(single-pass extraction)** — 대화가 오가는 동안 사실을 한 번의 처리로 추출해 저장한다. 이때 사용자 발화뿐 아니라 **에이전트가 생성·확인한 사실**도 일급 데이터로 취급해 함께 저장한다("사용자가 X라고 했다"와 "에이전트가 X를 확인했다"를 나란히 기록).
2. **다중 신호 검색(multi-signal retrieval)** — 질문이 오면 세 가지 점수를 병렬로 매긴다: ① 의미 유사도(semantic), ② 키워드 매칭(keyword), ③ 개체 매칭(entity). 세 신호를 결합해 가장 관련된 기억만 골라 컨텍스트에 넣는다.

이 구조가 정확도를 끌어올리는 지점은 **시간 추론(temporal)**과 **다단계 추론(multi-hop)**이다. 단순 의미 검색만으로는 "언제, 무엇 다음에" 같은 순서·시점 질문이나 "A를 거쳐 B로 이어지는" 연결 질문에 약한데, 개체·키워드 신호를 더하면 이런 질문의 정답률이 크게 오른다.

## 구체 예시·사례

Mem0가 공개한 수치를 보자. 토큰 효율 알고리즘은 LoCoMo에서 **92.5점을 질의당 6,956토큰**으로 달성했다. 이전 방식이 대화당 약 26,000토큰을 쓰던 것과 비교하면 토큰을 1/4 수준으로 줄인 것이다. LongMemEval에서는 94.4점(6,787토큰/질의), 초장문 벤치마크 BEAM에서는 1M 토큰 규모 64.1점, 10M 규모 48.6점을 기록했다. 개선 폭이 가장 큰 영역은 시간 추론 +29.6점, 다단계 추론 +23.1점이었다 — 즉 "순서와 연결"을 요구하는 어려운 질문에서 이득이 집중됐다.

## 비슷한 것과 비교

| 방식                         | 정확도                 | 질의당 토큰               | 언제 쓰나              |
| ---------------------------- | ---------------------- | ------------------------- | ---------------------- |
| Full-context(통째로 주입)    | 대화 짧으면 높음       | 매우 큼(대화 길이에 비례) | 단기·소량 대화         |
| 단순 RAG 메모리(의미 검색만) | 중간                   | 중간                      | 사실 회상 위주         |
| 토큰 효율 메모리(다중 신호)  | 높음(특히 시간·다단계) | 작음(약 7K)               | 장기·다세션, 비용 민감 |

선택 기준: 대화가 여러 세션으로 길어지고 시점·연결 추론이 필요하며 비용이 문제라면 다중 신호 메모리, 짧고 일회성이면 굳이 메모리 계층 없이 full-context가 단순하고 낫다.

## 왜 지금 중요한가

- Mem0의 2026년 4월 토큰 효율 메모리 알고리즘은 LoCoMo 92.5점을 질의당 6,956토큰에 달성했고(이전 대비 약 4배 토큰 절감), 시간·다단계 추론에서 각각 +29.6·+23.1점을 기록했다. [State of AI Agent Memory 2026](https://mem0.ai/blog/state-of-ai-agent-memory-2026)
- LoCoMo 벤치마크(1,540문항, 단일·다단계·오픈도메인·시간 회상 4범주)와 열 가지 메모리 접근법의 첫 대규모 비교 연구는 ECAI 2025에서 발표됐다. [Context Engineering for AI Agents(Mem0)](https://mem0.ai/blog/context-engineering-ai-agents-guide)
- 2026년 들어 메모리가 시스템 프롬프트에 다 밀어 넣는 대상이 아니라 별도 계층(1급 아키텍처 요소)으로 자리 잡으면서, "정확도 대 토큰"의 정량 비교가 실무 선택 기준이 됐다.

## 회계법인 AI 직무 연결 포인트

감사·회계 AI는 본질적으로 다세션·장기 기억 문제다. 한 고객사의 전기 감사 조서, 지난 분기 질의응답, 담당자와의 이전 협의 내용을 다음 감사에서 정확히 소환해야 한다. LoCoMo류 벤치마크는 "우리 감사 챗봇이 작년에 확인한 회계처리 근거를 올해 제대로 기억하는가"를 정량 검증하는 틀을 제공한다.

토큰 효율은 비용·통제 양쪽에서 중요하다. 감사 문서는 방대해 full-context 주입은 질의마다 수만 토큰을 태우고, 그만큼 환각과 컨텍스트 로트(context rot) 위험도 커진다. 필요한 사실만 검색해 넣는 메모리는 비용을 낮추는 동시에, 답변 근거를 추적 가능한 소수의 기억 항목으로 좁혀 감사 증거의 추적성(traceability)을 높인다. 시간 추론 강화는 "언제 어떤 조정을 했는가" 같은 감사에서 잦은 시점 질문에 직접 도움이 된다.

## 핵심 용어·논쟁

- **LoCoMo** — 다세션 긴 대화 기반 메모리 벤치마크. 1,540문항, 단일·다단계·오픈도메인·시간 회상 4범주.
- **Full-context** — 과거 대화를 통째로 컨텍스트에 넣는 방식. 단순하지만 토큰 비용이 큼.
- **다중 신호 검색** — 의미·키워드·개체 세 점수를 결합해 기억을 검색.
- **시간 추론(temporal)/다단계 추론(multi-hop)** — 시점·순서 질문 / 여러 사실을 연결해야 답하는 질문.
- 논쟁: 벤치마크 점수가 실제 업무 신뢰성을 대표하느냐. LoCoMo 고득점이 특정 도메인(감사·의료)의 정확성을 보장하진 않으며, 도메인별 평가셋 구축의 필요성이 제기된다.

## 자료 깊이 읽기

### State of AI Agent Memory 2026 (Mem0) — 영어, 리서치 리포트, 중상
Mem0의 2026 메모리 현황 리포트를 직접 확인해 정리했다. LoCoMo(1,540문항 4범주)를 기준으로 열 가지 메모리 접근법(문헌 베이스라인·오픈소스 툴·RAG·full-context·OpenAI Memory·Zep 등)을 비교했고, 2026년 4월 알고리즘이 LoCoMo 92.5점을 6,956토큰/질의(이전 약 26K 대비 급감)로 달성했다고 보고한다. 핵심 설계는 단일 패스 추출(에이전트 확인 사실도 일급 저장)과 의미·키워드·개체 3신호 결합 검색이며, 이득은 시간(+29.6)·다단계(+23.1) 추론에 집중됐다.
