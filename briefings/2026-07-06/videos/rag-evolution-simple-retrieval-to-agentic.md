# 영상 정리: RAG's Evolution: From Simple Retrieval to Agentic AI

> 원본: [RAG's Evolution: From Simple Retrieval to Agentic AI](https://www.youtube.com/watch?v=JB2P5Gk23VI) — 영어, 유튜브, 입문, 약 3분
> 연결된 브리핑: [2026-07-06.md](../../2026-07-06.md)
> 이 문서는 영어 자막을 실제로 내려받아 확인하고 정리했다. 짧은 애니메이션 설명 영상으로, 정보 검색(Information Retrieval)의 역사를 4단계로 압축해서 보여준다.

## 정보 검색의 4단계 진화

```
[키워드 검색]
   역색인(inverted index)으로 "이 단어가 어디 있는가"만 매칭
   TF-IDF/BM25로 랭킹 → 언어를 "기호"로만 취급, 의미는 모름
        │  한계: 동의어·모호성·복합 의도를 못 잡음 (예: "python" = 코딩? 뱀?)
        ▼
[시맨틱 검색]
   텍스트를 벡터(고차원 숫자)로 표현 → 의미가 가까운 개념은 벡터 공간에서도 가까움
   (예: coffee와 espresso는 가깝고, house는 멀다)
   키워드 검색을 대체하지 않고 보완 → 하이브리드 검색 등장
        │  발전: 정확한 키워드가 없어도 "의도"를 파악해 문서를 찾음
        ▼
[전통적 RAG]
   LLM은 사실을 "기억"하지 않고 학습 데이터 패턴으로 다음 토큰을 예측할 뿐
   → 학습 시점 이후 정보·비공개 문서를 모름 (LLM의 근본 한계)
   해결책: 질문 → 외부 지식베이스 검색 → 검색 결과로 프롬프트 보강 → 답변 생성
   (오프라인에 문서를 벡터DB에 임베딩 → 질의 시 1회 검색 → 모델에 그대로 전달, 선형 파이프라인)
        │  한계: 파이프라인이 고정(predetermined)되어 있어 새로운 상황에 적응 못함
        │        "답변은 검색 품질만큼만 좋다" — 결국 전통적 검색의 한계로 회귀
        ▼
[정교화된 RAG]
   리랭커(reranker) 추가로 결과 재정렬, 쿼리 재작성/확장으로 재현율 향상
   하이브리드 검색(키워드+시맨틱)이 표준이 됨
   → 더 정확해졌지만 여전히 "정적"(파이프라인 구조 자체는 고정)
        │
        ▼
[에이전틱 RAG]
   에이전트 = LLM + 도구를 활용해 자율적으로 작업을 수행하는 시스템
   질문이 들어오면 에이전트가 스스로 판단:
     - 검색이 필요한가?
     - 어디를 검색할 것인가?
     - 어떤 질문을 던질 것인가?
     - 충분한 정보를 얻었는가?
   → 출처 비교, 주장 검증, 쿼리 정제·반복(iterate)까지 스스로 수행
   → API 호출, 다중 지식베이스 조회, 멀티모달 데이터 통합 가능
   → 검색이 고정 단계가 아니라 "추론 과정의 일부로 호출되는 도구"가 됨
```

## 이 영상의 결론 문장

"단순 검색에서 현재의 에이전틱 RAG까지, 우리가 반복해서 배운 것은 — 다음 큰 도약은 더 나은 답이 아니라, **답을 찾는 방법을 아는 시스템**이라는 것이다. AI에서 가장 어려운 부분은 생성(generation)이 아니라, **무엇을 봐야 할지 결정하는 것**이다."

## 왜 이 짧은 영상이 다른 두 영상(오케스트레이션 강의)과 이어지는가

이 영상의 "에이전틱 RAG"는 앞서 정리한 두 오케스트레이션 강의의 **쿼리 분해(Query Decomposition) 에이전트**·**ReAct 패턴**과 사실상 같은 개념을 가리킨다. 검색을 "고정된 파이프라인 1단계"로 보던 시각에서, "에이전트가 추론 중에 필요하면 호출하는 도구 중 하나"로 보는 시각으로 전환된 것이 핵심이며, 이는 Tool Use 패턴이 RAG라는 구체적 도메인에 적용된 사례로 이해할 수 있다.
