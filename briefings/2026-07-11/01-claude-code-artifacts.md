# 오늘의 AI 개념: 클로드 코드 아티팩트(Claude Code Artifacts)

> 작성일: 2026-07-11 · 분류: claude-code

## 한 줄 정의

클로드 코드가 터미널 작업 결과를 팀원이 브라우저에서 바로 볼 수 있는 웹 페이지로 자동 변환해 공유하는 기능이다.

## 쉬운 설명

클로드 코드(Claude Code)는 원래 터미널 안에서 텍스트로 대화하며 코드를 작성하는 도구다. 작업 결과가 표나 차트, 여러 옵션 비교처럼 텍스트로 읽기 불편한 형태일 때가 많은데, 이런 결과를 팀원에게 보여주려면 스크린샷을 찍거나 별도로 정리해야 했다.

아티팩트는 이 과정을 없앤다. 클로드가 세션 중 만든 결과물을 HTML이나 마크다운 파일로 작성한 뒤, claude.ai의 비공개 URL로 곧바로 게시한다. 비유하면 회의 중 화이트보드에 그린 그림을 즉석에서 사진 찍어 단체 채팅방에 올리는 것과 비슷하다. 팀원은 링크만 열면 되고, 세션이 계속되는 동안 페이지도 실시간으로 갱신된다.

기존에 claude.ai 대화창에서 쓰던 아티팩트 기능과는 성격이 다르다. claude.ai의 아티팩트는 대화 하나에 종속된 결과물 미리보기에 가까웠다면, 클로드 코드 아티팩트는 코딩 세션에서 나온 작업물을 조직 내 동료와 공유하는 협업 문서에 가깝다.

## 동작 원리

클로드는 세션에서 만든 결과를 프로젝트 안의 HTML 또는 마크다운 파일로 작성한다.

파일을 게시하기 전에 사용자에게 권한을 요청한다. 예를 들어 "Claude wants to publish 'Deploy failures by service'..."라는 안내가 뜨고, 사용자가 승인하면 그때 claude.ai의 전용 URL로 업로드된다.

업로드된 페이지는 `*.claudeusercontent.com` 도메인의 샌드박스에서 렌더링되며, 외부 스크립트·스타일시트·API 호출을 막는 엄격한 콘텐츠 보안 정책(CSP, 웹사이트가 실행할 수 있는 리소스를 제한하는 보안 규칙)이 적용된다. 세션이 이어지며 파일이 다시 게시되면 같은 URL이 새 버전으로 갱신되고, 팀·엔터프라이즈 요금제에서는 이전 버전도 선택해 볼 수 있다.

## 구체 예시·사례

개발자가 풀 리퀘스트 리뷰어에게 변경 사항을 설명해야 하는 상황을 생각해보자. "이 PR을 diff에 주석을 달아 설명하는 아티팩트를 만들어줘"라고 요청하면, 클로드는 변경된 코드 옆에 각 수정의 이유를 적은 한 페이지짜리 웹 페이지를 만들어 게시한다.

리뷰어는 슬랙에 붙여넣은 긴 텍스트 설명 대신 링크 하나를 받는다. 페이지를 열면 diff와 주석이 나란히 배치되어 있어, 코드를 따라 읽으며 맥락을 재구성할 필요가 없다.

## 비슷한 것과 비교

| 구분 | claude.ai 아티팩트 | 클로드 코드 아티팩트 | 정적 사이트 배포(예: GitHub Pages) |
|---|---|---|---|
| 생성 주체 | 대화창의 클로드 | 코딩 세션의 클로드 | 개발자가 직접 빌드·배포 |
| 공유 범위 | 대화 참여자 | 조직 내 동료(팀·엔터프라이즈) | 설정에 따라 공개 가능 |
| 백엔드 | 없음 | 없음(정적 페이지만) | 별도 구성 가능 |
| 용도 | 대화 결과 미리보기 | 세션 작업물 공유·협업 | 지속 운영되는 웹사이트 |

셋 다 "웹 페이지 배포"라는 점은 같지만, 클로드 코드 아티팩트는 백엔드 없는 1회성 정적 페이지로 세션 결과를 빠르게 공유하는 용도에 최적화되어 있다는 점에서 선택 기준이 갈린다.

## 왜 지금 중요한가

클로드 코드 공식 문서는 아티팩트가 Pro·Max·Team·Enterprise 요금제에서 `/login`으로 로그인한 세션에 한해 제공된다고 명시한다. ([Share session output as artifacts](https://code.claude.com/docs/en/artifacts))

같은 문서는 이 기능이 Anthropic API를 사용하는 세션에서만 동작하며, Amazon Bedrock·Google Vertex AI·Microsoft Foundry를 경유하는 세션에서는 지원되지 않는다는 제약도 밝히고 있다. ([Share session output as artifacts](https://code.claude.com/docs/en/artifacts))

에이전틱 코딩 도구가 텍스트 출력을 넘어 시각적 산출물을 직접 배포하는 방향으로 확장되고 있다는 점에서, 이번 업데이트는 코딩 에이전트가 협업 도구로 진화하는 흐름을 보여주는 사례다.

## 회계법인 AI 직무 연결 포인트

감사팀이 데이터 분석 결과를 파트너나 클라이언트에게 보고할 때, 클로드 코드로 이상거래 탐지 결과를 분석한 뒤 아티팩트로 대시보드를 만들어 링크 하나로 공유할 수 있다. 엑셀 파일을 첨부하는 대신 브라우저에서 바로 확인 가능한 요약 화면을 전달하는 방식이다.

감사 증적을 정리하는 과정에서도 활용 여지가 있다. 특정 계정과목의 조사 과정을 아티팩트로 계속 갱신하면, 진행 상황을 실시간 타임라인 형태로 검토자에게 보여줄 수 있다.

다만 감사 데이터는 민감정보인 경우가 많아, 조직 정책상 CMEK(고객관리형 암호화 키)나 제로 데이터 리텐션이 설정된 환경에서는 아티팩트 기능 자체가 비활성화된다는 점을 실무 도입 전에 확인해야 한다.

## 핵심 용어·논쟁

- 아티팩트(Artifact) — 클로드 코드가 세션에서 게시하는, 백엔드 없는 1회성 정적 웹 페이지.
- CSP(콘텐츠 보안 정책) — 외부 스크립트·API 호출 등을 차단해 페이지가 안전하게 렌더링되도록 제한하는 보안 규칙.
- 제로 데이터 리텐션(Zero Data Retention) — 세션 데이터를 서버에 남기지 않는 조직 정책으로, 이 정책이 켜지면 아티팩트 기능이 꺼진다.
- 준수 API(Compliance API) — 조직이 게시된 아티팩트 목록·버전을 조회하고 삭제할 수 있는 관리자용 API.

아티팩트는 게시·공유·삭제 이력이 `claude_artifact_*` 감사 로그로 남는다는 점에서 통제 가능한 기능으로 설계되었지만, 조직 외부로는 절대 공유할 수 없다는 제약이 있어 외부 이해관계자(예: 감사 대상 회사)에게 결과를 보여줘야 하는 경우에는 HTML 파일을 직접 전달하는 우회가 필요하다.

## 자료 깊이 읽기

### Share session output as artifacts — 영어, 공식 문서, 중급
클로드 코드 공식 문서로, 아티팩트가 무엇인지부터 생성·갱신·공유 절차, 페이지 제약(외부 요청 차단, 백엔드 없음, 16MiB 크기 제한), 요금제별 이용 조건, 조직 관리자 설정까지 전 과정을 다룬다. "무엇을 언제 만들지" 프롬프트 예시가 풍부해 실무 적용 방법을 바로 참고할 수 있다. 감사 로그 이벤트 타입과 준수 API 엔드포인트까지 명시되어 있어 보안 검토 시에도 근거 자료로 쓸 수 있다. 별도 영상 없이 텍스트만으로 기능 전체를 이해할 수 있는 가장 신뢰도 높은 자료다.

### Claude Code now supports artifacts — 영어, 공식 블로그, 초급
Anthropic 공식 블로그의 발표 글로, 아티팩트 기능이 왜 만들어졌는지와 PR 워크스루·대시보드·체크리스트 등 활용 예시를 소개한다. 공식 문서보다 가볍게 기능의 취지를 파악하기 좋다.
