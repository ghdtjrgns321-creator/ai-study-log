# 브리핑 출력 형식

새 브리핑은 아래 형식을 따른다. 섹션 제목·순서를 바꾸지 않는다.
단, `동작 원리`·`구체 예시·사례`·`비슷한 것과 비교`·`핵심 용어·논쟁`은 **해당할 때만** 넣는다(억지로 채우지 않는다). 최소 3개는 포함한다.
파일 경로: `briefings/YYYY-MM-DD/NN-슬러그.md` (NN은 그날 순번 01~05).

# 오늘의 AI 개념: {개념 이름}(영문 병기)

> 작성일: YYYY-MM-DD · 분류: {claude-code | ai-concept | audit | trend}

## 한 줄 정의

한 문장으로 핵심을 정의한다. 비전문가도 이해할 수 있게.

## 쉬운 설명

3~4문단. 비유를 하나 넣어 직관적으로 풀되, 정확성을 해치지 않는다.
기존 유사 개념과의 차이를 한 문단으로 짚는다.

## 동작 원리

이 개념이 내부적으로 **어떻게 작동하는지** 단계별로 푼다. "무엇을 한다"가 아니라 "어떤 순서로, 무엇을 근거로 그렇게 되는지"까지.
구조가 있으면 텍스트 도식(화살표·번호 흐름)이나 간단한 표로 시각화한다. 코드·설정이 핵심이면 짧은 스니펫을 넣는다.

## 구체 예시·사례

추상 설명 대신 **실제 상황 하나**를 끝까지 따라간다. 입력→처리→출력을 구체적인 값·시나리오로 보여준다.
가능하면 숫자·코드·실제 도구 이름을 쓴다. "이런 경우 이렇게 동작한다"를 독자가 눈으로 그릴 수 있게.

## 비슷한 것과 비교

혼동하기 쉬운 인접 개념·경쟁 도구를 **표로 비교**한다. 축은 목적·강점·한계·언제 쓰나 등.
"그래서 무엇을 언제 골라야 하는가"라는 선택 기준을 한 줄로 정리한다. (비교 대상이 없으면 이 섹션 생략)

## 왜 지금 중요한가

최근(가급적 6개월 이내) 근거·수치·동향을 든다. 각 주장에는 출처 링크를 붙인다.
출처 없는 주장·지어낸 수치는 쓰지 않는다.

- [출처 제목](URL)

## 회계법인 AI 직무 연결 포인트

이 개념이 회계·감사 실무(감사 증거, 지속적 감사, 이상거래 탐지, 규정 문서 처리 등)에
어떻게 쓰이는지 2~3문단. 사용자가 회계·감사 도메인 학습자임을 전제로 한다.

## 핵심 용어·논쟁

이 주제를 이해하려면 알아야 할 용어 3~5개를 짧게 정의한다(용어 — 한 줄 뜻).
현재 진행 중인 찬반·미해결 쟁점이 있으면 한 문단으로 정리한다. (쟁점이 없는 성숙 주제는 용어 정의만)

## 자료 깊이 읽기

가장 좋은 자료 **2~3개를 골라 요약**한다. 텍스트 자료는 본문에 5~8줄로 요약한다.
**유튜브 링크는 별도 상세 요약 파일로 뺀다** — 본문에는 3~5줄 소개 + 링크만 두고, 실제 상세 요약(영상 전체를 다룬 study-note 수준, 분량 제한 없음)은 `briefings/YYYY-MM-DD/videos/{슬러그}.md`에 작성한다.
- 요약은 반드시 **실제 내용을 확인한 자료만** 쓴다. 자막·본문을 못 가져왔으면 지어내지 말고 `(요약 불가 — 자막/본문 확인 실패)`로 표시한다. 유튜브 자막은 `yt-dlp --write-auto-sub --write-sub --sub-lang ko,en`으로 직접 내려받아 확인한다(WebFetch로는 자막을 못 가져온다).
- 나머지 참고 자료는 요약 없이 링크만 나열한다.

### 유튜브 상세 요약 파일(`videos/{슬러그}.md`) 작성 규칙
- 파일 맨 위에 원본 링크·채널/재생시간·연결된 브리핑 경로·"자막을 실제로 내려받아 확인했다"는 출처 문구를 표기한다.
- 영상이 다루는 하위 주제·챕터·팁 목록을 **빠짐없이** 순서대로 정리한다(압축·생략 금지 — 이 파일의 존재 이유가 "본문보다 자세히"이다).
- 파이프라인·아키텍처·오케스트레이션 패턴처럼 데이터/흐름이 있는 내용은 ASCII 다이어그램으로 흐름을 먼저 보여주고 산문으로 해설한다(diagram-driven-docs 스킬 참고).
- 실무 팁·명령어·수치는 원문 그대로 보존하고, 자막 인식 오류로 불확실한 부분은 "직접 재확인 필요"라고 표시한다.
